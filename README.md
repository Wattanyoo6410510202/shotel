# Vue 3 + TypeScript + Vite

This template should help get you started developing with Vue 3 and TypeScript in Vite. The template uses Vue 3 `<script setup>` SFCs, check out the [script setup docs](https://v3.vuejs.org/api/sfc-script-setup.html#sfc-script-setup) to learn more.

Learn more about the recommended Project Setup and IDE Support in the [Vue Docs TypeScript Guide](https://vuejs.org/guide/typescript/overview.html#project-setup).
เรียบร้อยครับ! ผมได้ติดตั้งระบบโปรโมชั่นพร้อมฐานข้อมูล SQLite และระบบ Admin ให้คุณแล้ว โดยมีรายละเอียดดังนี้ครับ:

  วิธีใช้งานระบบ Admin
   1. การเข้าสู่ระบบ: ให้คุณพิมพ์ ?admin=true ต่อท้าย URL ของเว็บ (เช่น localhost:5173/?admin=true) หน้าจอ Login ของ Admin จะเด้งขึ้นมา
   2. รหัสผ่าน: ใช้รหัส 1234 ตามที่กำหนด
   3. การอัปเดต: เมื่อ Login แล้ว คุณสามารถเลือกไฟล์รูปภาพและพิมพ์คำอธิบายโปรโมชั่นใหม่ได้เลย เมื่อกด Save ระบบจะบันทึกลง SQLite และรีเฟรชหน้าเว็บให้เห็นผลทันที

  ส่วนประกอบของระบบ
   * Backend (Express + SQLite): อยู่ในโฟลเดอร์ server/ จัดการเรื่องการเก็บข้อมูลและไฟล์รูปภาพที่อัปโหลด (เก็บไว้ใน server/uploads/)
   * Promotion Modal: เมื่อคนทั่วไปเข้าเว็บ จะมี Popup แสดงภาพและข้อความโปรโมชั่นล่าสุดเด้งขึ้นมาทันที (จะเด้งแค่ครั้งเดียวต่อการเปิด Browser เพื่อไม่ให้รบกวนผู้ใช้)
   * Database: ใช้ SQLite ไฟล์ server/database.sqlite ดูแลง่ายและไม่ต้องติดตั้ง Database Server เพิ่มเติม

  วิธีรันระบบ (สำคัญ)
  เนื่องจากตอนนี้มีทั้งส่วนหน้าเว็บ (Frontend) และหลังบ้าน (Backend) คุณต้องรัน 2 คำสั่งพร้อมกัน:
   1. รันหน้าเว็บ: npm run dev
   2. รันหลังบ้าน: npm run server (ผมได้เพิ่ม script นี้ลงใน package.json ให้แล้วครับ)

  ลองใช้งานดูได้เลยครับ ถ้าอยากให้ปรับดีไซน์ Popup หรือเพิ่มส่วนไหนบอกได้เลยนะครับ!

  ตอนนี้ใช้ npm start เพื่อรัน express

  ปัจจุบันระบบ Backend ที่ผมสร้างให้คุณใช้เทคโนโลยีที่เน้น "ความเรียบง่ายแต่ทรงพลัง" โดยมีรายละเอียดดังนี้ครับ:

  1. Backend ของคุณใช้อะไรอยู่บ้าง? (The Stack)
   * Node.js & Express: เป็นตัวรัน Server หลักที่ได้รับความนิยมที่สุดในโลก JS เพราะทำงานได้เร็วและเบา
   * SQLite: เป็นฐานข้อมูลที่เป็น "ไฟล์เดียว" (database.sqlite) ไม่ต้องติดตั้งโปรแกรมฐานข้อมูลแยกให้วุ่นวาย แต่สามารถเก็บข้อมูลและค้นหาได้เหมือนฐานข้อมูลใหญ่ๆ (MySQL/PostgreSQL)
   * Multer: เป็น Library ที่ทำหน้าที่จัดการ "การรับไฟล์" (เช่น รูปภาพโปรโมชั่นที่คุณอัปโหลด) แล้วนำไปวางในโฟลเดอร์ uploads ให้เรา
   * CORS: ตัวจัดการความปลอดภัยที่อนุญาตให้หน้าเว็บ (Frontend) คุยกับหลังบ้าน (Backend) ได้

  2. อธิบายการทำงาน (How it works)
   * การเก็บข้อมูล: เมื่อคุณกด Save โปรโมชั่น หลังบ้านจะรับไฟล์รูปไปเก็บไว้ในโฟลเดอร์ uploads และบันทึก "ชื่อไฟล์" กับ "คำอธิบาย" ลงในตาราง promotion ใน SQLite
   * การดึงข้อมูล: ทุกครั้งที่มีคนเข้าเว็บ หน้าบ้านจะส่งคำขอ (Request) ไปที่ GET /api/promotion/latest หลังบ้านก็จะไปควักเอาโปรโมชั่นอันล่าสุดจาก SQLite ออกมาส่งกลับไปให้หน้าบ้านแสดงผล
   * ระบบ Admin: ทุกคำสั่งที่เกี่ยวกับการ แก้ไข/ลบ จะต้องแนบรหัสผ่าน 1234 ไปด้วยเสมอ เพื่อป้องกันคนแอบเข้ามาแก้ข้อมูลครับ

  ---

  3. กรณีขึ้น Production (การใช้งานจริงบนอินเทอร์เน็ต)
  ถ้าคุณจะเอาเว็บนี้ไปรันจริงๆ บน Server ที่มีชื่อโดเมน (เช่น www.shatyai.com) มีสิ่งที่ต้องทำดังนี้ครับ:

  ก. การจัดการไฟล์ (Deployment)
   * Build Frontend: คุณต้องรัน npm run build เพื่อแปลงโค้ด Vue ให้เป็นไฟล์ HTML/CSS/JS นิ่งๆ (Static files)
   * Merge Server: แก้ไข server/index.js ให้ทำหน้าที่ "ส่งไฟล์หน้าเว็บ" ออกไปด้วย (Serve static files) เพื่อให้รันแค่ Server ตัวเดียวแต่เห็นทั้งเว็บและหลังบ้าน

  ข. การเลือก Hosting
   * VPS (เช่น DigitalOcean, AWS, Linode): เป็นวิธีที่แนะนำที่สุดสำหรับ Node.js + SQLite เพราะคุณจะได้เครื่องคอมพิวเตอร์ออนไลน์มา 1 เครื่อง ซึ่งเราสามารถเขียนไฟล์ลงไปได้ (SQLite ต้องการสิทธิ์ในการเขียนไฟล์)
   * Platform (เช่น Render, Railway): ใช้งานง่ายมาก แค่ต่อกับ GitHub ก็รันได้เลย แต่ ต้องระวัง เรื่องการเก็บไฟล์รูปภาพและฐานข้อมูล เพราะพวกนี้มักจะลบไฟล์ทิ้งทุกครั้งที่อัปเดตโค้ดใหม่ (ต้องใช้สิ่งที่เรียกว่า Persistent Storage Disk)

  ค. เครื่องมือที่ต้องใช้เพิ่ม
   * PM2: เป็นโปรแกรมที่จะคอยเฝ้า Server ของคุณ ถ้า Server มีปัญหาจนดับไป PM2 จะทำการชุบชีวิต (Restart) ให้โดยอัตโนมัติ 24 ชม.
   * Nginx: ทำหน้าที่เป็น "ด่านหน้า" คอยจัดการเรื่อง HTTPS (รูปกุญแจเขียวๆ บน URL) และจัดการความปลอดภัยเบื้องต้น

  ง. ความปลอดภัย (Security)
   * รหัสผ่าน: ควรเปลี่ยนรหัส 1234 และเก็บไว้ในไฟล์ .env (Environment Variable) แทนการเขียนลงในโค้ดตรงๆ
   * HTTPS: ต้องทำ SSL เพื่อให้การส่งข้อมูล (รวมถึงรหัสผ่าน Admin) มีการเข้ารหัสครับ

  สรุป: ระบบตอนนี้พร้อมมากสำหรับการพัฒนาและทดสอบครับ และถ้าวันหนึ่งคุณต้องการขึ้น Production จริงๆ ผมสามารถช่วยปรับโค้ดให้พร้อมสำหรับขั้นตอนที่ผมอธิบายด้านบนได้ทันทีครับ!
